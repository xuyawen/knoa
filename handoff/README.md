# 知海 Knoa · 设计交付素材包（handoff）

给 AI 开发「知海 Knoa」跨境运营知识库应用时，按下面分工使用本文件夹内的素材。**不要只靠 PNG 让 AI 反推界面**——位图没有精确数值，还原度无法保证。

## 文件夹内容

| 文件 | 格式 | 给谁用 | 说明 |
|------|------|--------|------|
| `design-spec.md` | Markdown | **AI 主读** | 从 ardot 设计稿提取的设计规格：亮/暗双主题色值、字体、间距尺度、组件树、桌面主工作台页面结构、后端接口签名。这是最高保真的"图纸"，AI 应逐条对标。 |
| `2_3.svg` | SVG 矢量 | AI 读 + 人眼核对 | 亮色桌面版（1440×900）。含精确坐标、色值、圆角、文字。AI 可读取几何，定位间距/对齐。 |
| `6_2.svg` | SVG 矢量 | AI 读 + 人眼核对 | 暗色桌面版（1440×900）。 |
| `5_2.svg` | SVG 矢量 | AI 读 + 人眼核对 | 亮色移动端（390×844）。 |
| `5_126.svg` | SVG 矢量 | AI 读 + 人眼核对 | 暗色移动端（390×844）。 |
| `../exports/*.png` | PNG 2x | 仅人眼视觉核对 | 高清位图，用于你肉眼验收"长得对不对"，**不要**交给 AI 做数值反推。 |
| `../exports/*.pdf` | PDF | 分享/汇报 | 四版合并预览。 |

## 给新 AI 工作空间的喂料建议（保真度从高到低）

1. **最强：直接复用 `knoa/` 代码仓库** —— 前端 Vue 3 + Vite + TS（三栏布局 + 亮暗主题已还原）、后端 FastAPI 骨架（问答流式 mock + 知识库接口）都已就绪。把整个 `knoa/` 指给新工作空间，AI 在已有代码上继续填 RAG 即可，比从零看图写快且准。
2. **次强：本包 `design-spec.md` + 4 个 `.svg`** —— 若新工作空间是纯净环境、要从设计稿落地，让 AI 先读 `design-spec.md` 拿全部数值，再对照 `.svg` 校准布局细节。
3. **兜底：ardot 源文件（需新环境装 ardot MCP）** —— 文件 ID `702429835416563`，节点树含每个元素的精确 `fills`/`strokes`/`x,y,w,h`。若开发 AI 能连 ardot，直接读节点树是最无损的方式。
4. **不推荐：只给 PNG** —— 位图无数值，AI 必须猜测每个像素，亮暗主题的色差、间距、字号都容易漂。
